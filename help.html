Jeckyll

So Jeckyll is a static website generator, so it's the bit that actually reads the code to piece together the site. It's a pain in the arse to get onto your computer (at least it was on my computer, which was from work so super locked down) but once it's running it allows you to make edits to the code locally (i.e. in somewhere like Atom), run the website on your computer, and see the changes you've made.

If you'd like a go at installing Jeckyll, here are the steps. IMPORANT: This is much harder if you're running on a work computer as they've locked everything down securely so you can't install things!!

1) Go to the Kunji Lab Github Repo and download it as a zipfile (click on the green "clone or download" button). Extract the zipfile and drag it to the Desktop.

2) Set up a Ruby environment using instructions found at this link. These commands need to run in the terminal, which all the Macs should have. The terminal should be Bash if it's a home computer or -tsch if it's a work one.

3) Change your current directory in the terminal to the folder in the desktop with our website in using the command cd ~/Desktop/kunji-lab.github.io

4) Run the command gem install jekyll bundler

5) Run the command jekyll serve. This will give you a few lines of text, one of which will say "Server address: http://127.0.0.1:4000/". This is the location the website is currently running on, using your comptuer as a server! So go and type http://127.0.0.1:4000/ into your browser and it should come up!

6) Enjoy the fact you've succeeded in doing something that's really, really difficult to set up!

Congratulations! Jekyll is now building the website locally! Now, if you want you edit something, open up the kunji-lab.github.io folder in a text editor, such as Atom, to see the code. There you can edit text, images, etc. If you then save what you've been working on, and refresh the page in your browser, you should see the new changes appear! Don't worry, this is just local for now - it's not the live website you're changing. Play around with it and have fun!

I've made local changes to a page that I'd like to go on the actual website! How do I do this?

Great question! This is where we use github: We're going to get Github to look at our work and identify differences between our work and what it currently has stored. Then we will get Github to take those changes and save them. Finally, we'll push our changes to the repo. Let's go!



1) Go back into the terminal, and ensure that the current directory is kunji-lab.github.io

2) Run the command git add . in the terminal. This identifies changes.

3) Run the command git commit -m "commit message in speech marks" in the terminal. This stores the changes with a message about what you've done (e.g. updated structural page). You'll need your github username and password for this!

4) Run the command git push in the terminal. This pushes the changes into our repo. The changes will occur to the live website in a few minutes!

This whole Jeckyll thing is difficult! Can I edit the website any other way?

Sure! The easiest thing to do would be to send what you'd like doing to me (Alannah) e.g. "Here's the writing I'd like to go onto the Cardiolipin project page" and I'll pop it on. Alternatively, you can use Github itself!



1) Go to the Kunji Lab Github Repo and click on the Fork button in the top right. This will create your own fork of the repository, and you'll see the name of the repository as, for exmaple, ak2022/kunji-lab. This means it's now a copy in your Github!

2) Click on the folders/files to access them, and once inside a file click on the pencil button on the top right to edit it. Make any changes, then click "Commit Changes" (essentially, fancy save).

3) When you're happy with the changes you've made, click on the "Pull Request" tab (Or go to github.com/ak2022/kunji-lab.github.io/pulls, for example). Click "Create New Pull Request" and give it a name, and write some info describing the changes you've made. Then click "Create Pull Request"

4) It's done! It will wait until I have reviewed it (to ensure it's all okay :D) and then I will merge it into the main website.
